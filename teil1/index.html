Testseite für walker.js (Custom Endpoint)

Hier wird eine lokale Fassung von walker.js geladen und im Code als Destination ein eigener Endpunkt und die Console definiert (hier geht es zu einer einer einfachen Version mit Default Verhalten). Der Endpunkt bekommt das komplette Event als JSON Payload gesendet, wie es auch im elbLayer zu finden ist. Dort kann dann eine Speicherung in einem lokalen Log, Parsing in Standardevents des server-side Google Tag Managers in Form eines eigenen Clients, das Versenken des Events in einer eigenen Datenbank o. Ä. passieren.

Weitere Hinweise zur Verwendung von walker.js siehe https://docs.elbwalker.com/tagging/basics.
